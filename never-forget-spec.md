# Never Forget — Specification

## Executive Summary

Telegram-бот для небольшого цветочного магазина в Валенсии. Решает проблему хаотичных продаж через личку Instagram: систематизирует заказы, автоматизирует напоминания о важных датах клиентов и даёт владелице CRM-инбокс прямо в Telegram. Целевая аудитория — 270 подписчиков существующего Telegram-канала + новые клиенты.

---

## Problem Statement

Владелица магазина делает до 5 букетов в день, работает одна с мужем-курьером. Каждый заказ — ручной диалог в Instagram: уточнение, согласование, реквизиты оплаты, договорённость о доставке. Нет истории клиентов, нет повторных продаж, нет напоминаний. Время уходит на рутину вместо работы с цветами.

---

## Success Criteria

- Время обработки одного заказа сокращается с ~15 мин диалога до ~2 мин (подтвердить + отправить реквизиты)
- Минимум 1 повторный заказ в месяц от клиентов, получивших напоминание о дате
- Все активные заказы видны в одном месте — без переключения между чатами
- Клиент может сделать заказ самостоятельно без переписки

---

## User Personas

**Клиент-мужчина** — покупает букеты для жены/мамы/коллеги по поводу. Плохо помнит даты, не знает какие цветы выбрать, хочет чтобы всё было просто.

**Клиент-женщина** — покупает себе раз в месяц или на особые случаи. Знает что хочет, ценит удобство.

**Владелица (Администратор)** — работает из дома, делает букеты сама. Принимает заказы, подтверждает оплату, координирует мужа-курьера.

---

## User Journey

### Клиент — первый визит

1. Открывает бот → видит приветствие на выбранном языке (RU / UA)
2. Онбординг: бот ведёт по шагам —
   - Имя
   - Важные даты (день рождения жены, мамы, годовщина, собственный ДР и т.д.) — можно добавить несколько
   - Предпочтения по цветам (розы, пионы, полевые, без предпочтений)
   - Примерный бюджет на букет (до 30€ / 30–60€ / 60€+)
3. Попадает в главное меню

### Клиент — оформление заказа из каталога

1. Меню → "Каталог букетов"
2. Листает букеты (фото + название + цена)
3. Нажимает "Заказать"
4. Выбирает: доставка / самовывоз
   - Доставка: вводит адрес → бот формирует Google Maps ссылку + желаемое время
   - Самовывоз: выбирает время
5. Выбирает способ оплаты: Bizum / банковская карта / наличные (только для известных клиентов)
   - Bizum/карта: бот показывает реквизиты, клиент платит вручную, нажимает "Оплатил"
   - Наличные: подтверждает при получении
6. Заказ создан → клиент видит подтверждение с деталями

### Клиент — кастомный заказ

1. Меню → "Заказать букет под себя"
2. Бот спрашивает по шагам: повод, бюджет, предпочтения по цветам, дата нужна
3. Заявка уходит владелице как задача
4. Она связывается через бот-инбокс

### Автоматические сообщения

- **За 3 дня до важной даты**: "Через 3 дня день рождения [имя]. Вот букеты в твоём бюджете:" → подборка из каталога
- **В день рождения клиента**: персональное поздравление от магазина

### Владелица — получение заказа

1. Уведомление в Telegram: "🆕 Новый заказ от [Имя]\nБукет: [Название] — 45€\nДоставка: завтра 14:00\nАдрес: [адрес] [Google Maps ссылка]\nОплата: Bizum (ожидает подтверждения)"
2. Открывает CRM-инбокс → находит клиента
3. Подтверждает оплату кнопкой → клиент получает "Оплата принята, готовим букет"
4. Может написать клиенту через инбокс (сообщение доставляет бот)

### Владелица — управление каталогом

1. Команда `/admin` → меню админа
2. Добавить букет: фото + название + цена
3. Удалить/скрыть букет (когда нет в наличии)

## Functional Requirements

### Must Have — P0

- [ ] Выбор языка при старте (RU / UA)
- [ ] Онбординг: имя, важные даты, предпочтения по цветам, бюджет
- [ ] Каталог букетов (фото, название, цена)
- [ ] Заказ из каталога: доставка/самовывоз, адрес, время, способ оплаты
- [ ] Показ реквизитов оплаты (Bizum / карта / наличные)
- [ ] Уведомление владелице о новом заказе (с Google Maps ссылкой)
- [ ] Подтверждение оплаты владелицей → уведомление клиенту
- [ ] Кастомный заказ: форма с шагами → задача для владелицы
- [ ] Напоминание за 3 дня до важной даты с подборкой по бюджету
- [ ] Поздравление клиента в его день рождения
- [ ] Админ-меню: добавить/убрать букет из каталога
- [ ] CRM-инбокс: список клиентов, карточка клиента (история, даты, предпочтения, заметки)
- [ ] Коммуникация владелицы с клиентом через бот (она пишет → бот доставляет)

### Should Have — P1

- [ ] История заказов клиента в его карточке
- [ ] Статусы заказа: новый → подтверждён → оплачен → в пути → выполнен
- [ ] Заметки владелицы по клиенту (свободный текст)
- [ ] Список всех активных заказов для владелицы (дашборд дня)
- [ ] Клиент может редактировать свои важные даты и предпочтения

### Nice to Have — P2

- [ ] Автоматические напоминания на ключевые праздники (8 марта, НГ, 14 февраля) — за неделю
- [ ] Статистика для владелицы: заказов в месяц, выручка, топ клиенты
- [ ] Возможность добавить несколько адресов доставки

## Technical Architecture

### Stack

- **Telegram Bot API** — python-telegram-bot или node-telegram-bot-api
- **Firebase Firestore** — база данных (новый проект)
- **Firebase Cloud Functions или Railway** — хостинг бота
- **Scheduler** — для отправки автоматических напоминаний (cron job)

### Data Model

```
users/
  {telegram_id}/
    name: string
    language: "ru" | "ua"
    phone: string (optional)
    preferences:
      flowers: string[]
      budget: "low" | "mid" | "high"
    dates:
      [{label: "День рождения жены", date: "MM-DD", name: "Оля"}]
    birthday: "MM-DD"
    notes: string (admin only)
    created_at: timestamp

bouquets/
  {id}/
    name: string
    price: number
    photo_id: string (Telegram file_id)
    available: boolean
    created_at: timestamp

orders/
  {id}/
    user_id: string
    bouquet_id: string | null (null = кастомный)
    type: "catalog" | "custom"
    status: "new" | "confirmed" | "paid" | "delivery" | "done"
    delivery_type: "delivery" | "pickup"
    address: string | null
    maps_url: string | null
    delivery_time: string
    payment_method: "bizum" | "card" | "cash"
    payment_confirmed: boolean
    custom_request: string | null
    price: number
    created_at: timestamp

messages/
  {id}/
    order_id: string | null
    from: "admin" | "client"
    user_id: string
    text: string
    created_at: timestamp
```

### System Components

1. **Bot Handler** — обрабатывает команды и callback-кнопки клиентов
2. **Admin Handler** — отдельные команды для владелицы (`/admin`)
3. **Scheduler** — ежедневно проверяет даты, отправляет напоминания (запускается в 10:00 по Валенсии)
4. **CRM Inbox** — система форвардинга сообщений между владелицей и клиентами через бота

### Integrations

- **Telegram Bot API** — единственная внешняя интеграция
- **Google Maps** — просто ссылка формата `https://maps.google.com/?q={адрес}`, никакого API не нужно
- Платежи — **ручные** (бот показывает реквизиты, человек платит сам)

### Security Model

- Владелица идентифицируется по `telegram_id` (хардкодится в конфиге как `ADMIN_CHAT_ID`)
- Клиенты — по `telegram_id`, никакой отдельной авторизации
- Firebase — закрытый доступ только с сервера (service account), правила Firestore блокируют прямой доступ
- Реквизиты оплаты хранятся в конфиге на сервере, не в базе

---

## Non-Functional Requirements

- Бот отвечает в течение 2 секунд
- Напоминания отправляются в 10:00 по мадридскому времени (UTC+2)
- До 50 активных пользователей — Railway free tier достаточно
- Фото букетов хранятся как Telegram file_id (не нужен отдельный storage)

---

## Out of Scope

- Автоматическая онлайн-оплата (Stripe, PayPal) — нет регистрации ИП
- Веб-панель администратора — только Telegram
- Многоязычность кроме RU/UA
- Интеграция с Instagram
- Аналитика и отчёты (P2, после MVP)
